# Nutrition Care Records System (NCRS) Frontend

A modern web frontend for the **Nutrition Care Records System (NCRS)**, built with **React**, **TypeScript**, and **Vite**. This application supports nutrition care workflows through a responsive and maintainable user interface.

---

## Overview

The NCRS Frontend is the client-side application for managing nutrition care records and related operations. It is designed for fast development, strong typing, and reliable deployments.

## Getting Started

### 1) Clone the repository

```bash
git clone https://github.com/edisontaimu9-ui/ncrs-frontend.git
cd ncrs-frontend
```

### 2) Install dependencies

```bash
npm install
```

### 3) Configure environment variables

Create a local `.env` file in the project root (do not commit real values):

```env
VITE_API_BASE_URL=https://api.example.com
VITE_APP_ENV=development
VITE_APP_NAME=Nutrition Care Records System
```

> In Vite, only variables prefixed with `VITE_` are exposed to client-side code.

### 4) Run the development server

```bash
npm run dev
```

The app typically runs at `http://localhost:5173`.

## Deployment (Vercel)

This project is configured for Vercel deployment.

### Steps

1. Import `edisontaimu9-ui/ncrs-frontend` into Vercel.
2. Add required environment variables in:
   - **Project → Settings → Environment Variables**
3. Deploy or redeploy the latest commit.

## Environment Variables Policy

- ✅ Keep real values in local environment or Vercel settings.
- ✅ Commit only `.env.example` with placeholders.
- ❌ Never commit real `.env` or secret values to GitHub.

## Troubleshooting

### App won’t start
- Ensure dependencies are installed with `npm install`.
- Check that required env vars exist and use the `VITE_` prefix.

### API requests fail
- Verify `VITE_API_BASE_URL` is correct.
- Confirm Vercel environment variables are set for the correct environment.

### Tailwind styles not applied
- Check:
  - `tailwind.config.js`
  - `postcss.config.js`
